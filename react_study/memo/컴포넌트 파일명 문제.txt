요약 :
React 컴포넌트의 파일명은 PascalCase를 따르기에 첫 글자를 대문자로 써야하며,
실수로 소문자로 쓰면 React가 컴포넌트임을 인식할 수 없다.

파일명 수정은 언제나 가능하지만, 문제가 생긴다면 그건 캐시 문제일테니 
브라우저의 캐시 삭제 기능과  Node.js의 캐시 삭제 명령어yarn cache clean 로 삭제해준뒤 
다시 React를 빌드해서 실행하면 된다.


----------------------------------------------------------------------------------------------------


파일명을 대문자로 작성해야 하는 경우는 React에서의 컴포넌트 네이밍 컨벤션입니다. 
일반적으로 React 컴포넌트의 파일명은 PascalCase를 따르며, 
이는 첫 글자를 대문자로 시작하는 것을 의미합니다. 
그래서 실수로 소문자로 파일명을 작성하면 React가 해당 파일을 컴포넌트로 인식하지 않을 수 있습니다.

파일명을 수정하는 것은 언제나 가능합니다. 
만약 파일명을 수정했음에도 불구하고 여전히 문제가 발생한다면, 다음과 같은 점들을 확인해볼 수 있습니다:

파일명 변경 후에도 여전히 문제가 발생하는 경우, 캐시 문제일 수 있습니다. 
이 경우 캐시를 제거하고 다시 빌드해 보세요.

파일명을 변경한 후에도 문제가 지속된다면, 
React 애플리케이션 자체에서 파일명을 참조하는 곳이 있는지 확인해야 합니다. 
예를 들어, import 문에서 해당 파일을 참조하는 부분이 있는지 확인해야 합니다.

파일명을 수정하는 것은 언제나 가능하며, 올바르게 수정하면 문제가 해결될 것입니다. 
하지만 일단 파일명을 수정한 후에도 문제가 지속된다면, 해당 문제의 원인을 찾아야 합니다.

캐시 삭제
캐시를 제거하는 방법은 다음과 같습니다.

브라우저 캐시: 개발 중인 React 애플리케이션이 브라우저에 의해 캐싱될 수 있습니다. 이 경우에는 브라우저 캐시를 지우고 페이지를 새로고침하면 됩니다. 대부분의 브라우저에서는 캐시를 지우는 기능을 제공합니다. 보통 "캐시 지우기" 또는 "데이터 지우기"와 같은 옵션을 찾을 수 있습니다.

Node.js 캐시: Node.js에서도 패키지 설치와 관련된 캐시가 있을 수 있습니다. 이 경우에는 npm cache clean 명령어를 사용하여 npm 캐시를 지울 수 있습니다.

bash
Copy code
npm cache clean --force
또는 yarn을 사용하고 있다면, yarn에 대한 캐시를 지우려면 다음 명령어를 사용합니다.

bash
Copy code
yarn cache clean
위의 명령어를 실행한 후에는 다시 React 애플리케이션을 빌드하고 실행해 보세요.
